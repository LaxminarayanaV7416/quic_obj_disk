Types of Storage Systems

File Storage

File storage is the most common and traditional way of storing data. In file storage data is stored in a hierarchical structure with files and folders. It is easy to use and manage, but it has limitations in terms of scalability and performance.

Block Storage

Block storage is a more advanced way of storing data. In block storage, data is divided into fixed-size blocks and each block is assigned a unique identifier. It provides better performance and scalability compared to file storage, but it requires more management and configuration.

Object Storage

Object storage is a modern way of storing data. In object storage, data is stored as objects with metadata and a unique identifier. It provides high scalability, durability, and cost-effectiveness. It is ideal for storing unstructured data such as images, videos, and documents.

Object Storage Current State

Object storage is widely used in cloud computing and big data applications. Some of the popular object storage systems are Amazon S3, Google Cloud Storage, and Microsoft Azure Blob Storage.

Oppurtunity

The proposed idea is to develop a QUIC based object storage system that can provide high performance, scalability, and security.

QUIC is a modern transport protocol that is designed to improve the performance of web applications. It provides low latency, multiplexing, and improved security features.

QUIC Introduction

  1. QUIC = Quick UDP Internet Connections
  2. Next-generation transport protocol developed by Google
  3. Foundation for HTTP/3 (the latest web protocol)
  4. Deployed across Google services (Chrome, YouTube, Gmail)

What QUIC Solves from HTTP2.0

  1. Head-of-Line Blocking: One lost packet stops ALL data streams
  2. TCP Handshake Overhead: Multiple round trips to establish connection
  3. Slow Connection Setup: 3 RTT for secure HTTPS connections
